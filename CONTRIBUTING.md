# 貢獻指南 (Contributing Guide)

歡迎來到 **Tsext Adventure: Halloween Haunt** 的貢獻指南！我們非常歡迎社區成員為這個搞笑瑟瑟文字冒險遊戲做出貢獻。

## 🎃 如何貢獻

### 1. Fork 專案
1. 點擊 GitHub 頁面上的 "Fork" 按鈕
2. 將專案複製到你的 GitHub 帳戶

### 2. 建立分支
```bash
git checkout -b add-new-pun
# 或
git checkout -b fix-story-bug
```

### 3. 進行修改
根據你想要貢獻的內容類型進行修改（詳見下方分類）

### 4. 提交變更
```bash
git add .
git commit -m "Add: 新的萬聖節雙關語"
git push origin add-new-pun
```

### 5. 建立 Pull Request
1. 在 GitHub 上建立 Pull Request
2. 詳細描述你的貢獻內容
3. 等待審查和回饋

## 📝 貢獻類型

### 🎭 故事內容 (Story Content)
- **新故事分支**: 在 `stories/halloween.json` 中添加新的場景和選擇
- **新結局**: 創建有趣的結局場景
- **角色對話**: 改進現有角色的對話內容

**範例格式**:
```json
{
  "new_scene": {
    "title": "新場景標題",
    "description": "場景描述...",
    "choices": [
      {
        "option": "A: 選擇描述",
        "next_scene": "next_scene_id"
      }
    ]
  }
}
```

### 😂 雙關語和幽默 (Puns & Humor)
- **新雙關語**: 在 `stories/common.json` 中添加萬聖節相關的雙關語
- **搭訕語句**: 添加創意的萬聖節搭訕語句
- **NSFW 幽默**: 添加成人幽默內容（需標記 NSFW）

**範例格式**:
```json
{
  "common_puns": [
    "新的萬聖節雙關語...",
    "另一個搞笑的雙關語..."
  ]
}
```

### 🐛 Bug 修復 (Bug Fixes)
- **故事邏輯錯誤**: 修復場景連接問題
- **JSON 格式錯誤**: 修正語法錯誤
- **遊戲流程問題**: 修復遊戲邏輯錯誤

### 🧪 測試 (Testing)
- **單元測試**: 在 `tests/` 中添加新的測試案例
- **故事完整性測試**: 確保所有場景都可以到達
- **邊界情況測試**: 測試異常輸入和邊界情況

### 📚 文件 (Documentation)
- **README 改進**: 更新專案說明
- **API 文件**: 添加程式碼註解
- **使用指南**: 創建詳細的使用說明

## 📋 貢獻規範

### 程式碼風格
- 使用 Python 3.8+ 語法
- 遵循 PEP 8 程式碼風格
- 添加適當的註解和文檔字串
- 使用有意義的變數和函數名稱

### 提交訊息格式
```
類型: 簡短描述

詳細描述（可選）

相關議題: #123
```

**類型**:
- `Add`: 新功能
- `Fix`: Bug 修復
- `Update`: 更新現有功能
- `Remove`: 移除功能
- `Docs`: 文件更新
- `Test`: 測試相關

### 測試要求
- 所有新功能必須包含測試
- 確保現有測試仍然通過
- 測試覆蓋率不應低於 80%
